# 🤖 AI Recommendation Docs

## AI 추천 방식

DoTheReco AI는 단순 시간 기반 일정 추천이 아니라,
사용자의 자연어 요청과 위치 정보를 함께 고려하여 추천 전략을 동적으로 변경합니다.

---

## 📍 기본 추천 기준

추천 엔진은 아래 요소를 함께 계산합니다.

* 고정 일정 사이의 빈 시간
* 현재 위치와 다음 일정 사이의 이동 거리
* 유동 일정 예상 소요 시간
* 일정 우선순위 및 마감일

이를 기반으로 하루 타임라인을 자동 생성합니다.

---

## 🚶 이동 최소화 전략

사용자가:

* "이동이 적은 순서로 짜줘"

와 같은 요청을 입력하면:

* 현재 위치 기준 가까운 장소를 우선 탐색
* 같은 지역의 유동 일정을 우선 배치
* 이동 시간이 가장 적은 후보 장소를 선택

예시:

* 건대 일정 이후 → 건대 근처 카페 추천
* 강남 일정 이후 → 강남역 주변 일정 추천

---

## 😴 여유로운 일정 전략

사용자가:

* "오늘 피곤해서 여유롭게 넣어줘"

와 같은 요청을 입력하면:

* 일정 사이 buffer 시간을 증가
* 이동 후 충분한 휴식 시간을 확보
* 결과적으로 하루에 배치되는 일정 수 감소

---

## ⏰ 마감 우선 전략

사용자가:

* "마감 가까운 일정 먼저 추천해줘"

와 같은 요청을 입력하면:

* D-Day가 가까운 일정 우선 정렬
* 우선순위(priority)가 높은 일정 우선 배치
* 시간이 부족할 경우 낮은 우선순위 일정 제외

---

## 🧠 자연어 전략 분석

사용자의 자연어 요청은 OpenAI API를 통해 분석되며,
추천 전략 객체로 변환되어 실제 추천 로직에 반영됩니다.

예시:

```json
{
  "minimizeMovement": true,
  "preferRest": false,
  "prioritizeDeadline": true,
  "bufferLevel": "medium",
  "priorityMode": "distance"
}
```

---

## 📍 위치 기반 장소 추천 방식

추천 엔진은 currentPlace 기반으로 다음 장소를 추천합니다.

흐름:

```text
현재 위치
→ 지역 키워드 추론
→ 장소 검색
→ 이동 거리 계산
→ 최적 장소 선택
```

예시:

```text
건국대학교 서울캠퍼스
→ 건대입구
→ 건대입구 올리브영 검색
```

이를 통해 현재 위치와 동선을 고려한 일정 추천을 구현했습니다.
